Shortest Paths

Shortest paths in graphs are a very important problem in many areas of computer science and other disciplines as well. The Visualisation-Tool provided resembles a graph in such a way that every square is a node inside the graph, which is connected to all sourrounding squares (nodes).

How does it work?

1. Select your algorithm; 2. Draw your obstacles; 3. Start the algorithm and see how it finds the shortest path from your set starting point to your set finishing point. Stop the algorithm to examine its current state, maybe draw some more obstacles and then resume execution.
Grey fields are fields the algorithm knows about, but didn't yet fully explore. Orange fields are the ones the algrithm already looked at.

Try it yourself

Interpretation of the grid

The grid resembles a graph, but you can also think of it as just a map. The difficulty of traveling between two squares is determined by their color. Every square of the grid is a node in a graph. All adjacent squares are reachable nodes in the corresponding graph.

Understand the weight function

The number between the two color-pickers resembles the difficulty of traveling from color one to color two.
The higher the number, the more difficult it is to travel from the square to the next. A black sqaure cannot be enetered.
You could also think of this map as a height map, where the darkness of a color resembles the height of the square you are standing on. The number displayed in the middle would then be the altitude you'd have to cross when crossing from one square to the next. A value of 0 is horizontal, anything smaller than 0 is going downhill (easy) and anything greater 0 is going uphill (difficult).
